**기본 전처리 (테스트셋 + 증강 비활성 훈련셋)**

```
[0,255] uint8 (H,W,C)
    → ToTensor()        → [0,1] float32, shape: (C,H,W)=(3,32,32)
    → Normalize(μ,σ)    → 채널별 z-score 정규화
```

CIFAR-10 채널별 통계값 (훈련셋에서 사전 계산된 값):

| 채널 | 평균 (μ) | 표준편차 (σ) |
|---|---|---|
| R | 0.4914 | 0.2470 |
| G | 0.4822 | 0.2435 |
| B | 0.4465 | 0.2616 |

정규화를 적용하면 각 채널의 픽셀값이 평균 0, 표준편차 1 근방으로 조정되어 학습이 안정화된다.

**Data Augmentation (훈련셋, Part G)**

과적합 방지를 위해 훈련 시 아래 두 가지 무작위 변환을 추가 적용한다.

```
RandomCrop(32, padding=4)   → 4픽셀 zero-padding 후 32×32 랜덤 크롭
RandomHorizontalFlip()      → 50% 확률로 좌우 반전
```

동일한 이미지가 매 epoch마다 다른 형태로 제시되므로 모델이 특정 위치나 방향에 과적합되는 것을 방지한다.

## 3. CNN 모델 구현 (Part A)

> **구현 원칙**: `torchvision.models`에서 제공하는 사전 정의 모델(VGG, ResNet 등)을 일절 사용하지 않고, `nn.Module`을 직접 상속하여 구현하였다.

### 3-1. 전체 아키텍처 설계 철학

하나의 `CNN` 클래스가 블록 명세 리스트(`blocks`)를 인수로 받아 다양한 구조를 생성하도록 설계하였다. 이를 통해 필터 수, 커널 크기, 블록 수, 풀링 유무 등을 코드 수정 없이 실험할 수 있다.

```python
class CNN(nn.Module):
    def __init__(self,
                 blocks: List[(out_channels, kernel_size, use_pool)],
                 conv_per_block: int = 1,   # VGG-style 중첩 conv
                 fc_dim: int = 256,
                 dropout: float = 0.0,
                 num_classes: int = 10,
                 in_channels: int = 3,
                 in_size: int = 32):
```

### 3-2. Baseline 모델 — 레이어별 상세 구조

기준 모델(baseline)은 3개의 합성곱 블록과 2개의 완전연결층으로 구성된다.

```
입력: (Batch, 3, 32, 32)
────────────────────────────────────────────────────
[Conv Block 1]
  Conv2d(3 → 32, kernel=3×3, padding=1)   출력: (B, 32, 32, 32)
  BatchNorm2d(32)
  ReLU
  MaxPool2d(2×2, stride=2)                출력: (B, 32, 16, 16)

[Conv Block 2]
  Conv2d(32 → 64, kernel=3×3, padding=1)  출력: (B, 64, 16, 16)
  BatchNorm2d(64)
  ReLU
  MaxPool2d(2×2, stride=2)                출력: (B, 64,  8,  8)

[Conv Block 3]
  Conv2d(64 → 128, kernel=3×3, padding=1) 출력: (B, 128, 8, 8)
  BatchNorm2d(128)
  ReLU
  MaxPool2d(2×2, stride=2)                출력: (B, 128, 4, 4)

[Classifier]
  Flatten                                  출력: (B, 2048)
  Linear(2048 → 256)
  ReLU
  Dropout(p=0.3)
  Linear(256 → 10)                         출력: (B, 10)
────────────────────────────────────────────────────
출력: (Batch, 10)  — 각 클래스의 로짓(logit) 값
```

### 3-3. 각 구성 요소의 역할

| 레이어 | 역할 | 선택 이유 |
|---|---|---|
| **Conv2d(k×k, padding=k//2)** | 커널을 이미지 위로 슬라이딩하며 지역 패턴(엣지, 텍스처, 형태) 추출 | padding=k//2 적용으로 공간 크기를 유지("same" 방식); 크기 축소는 MaxPool에만 위임 |
| **BatchNorm2d** | 미니배치 내 채널별로 평균·분산을 정규화하고, 학습 가능한 γ, β로 스케일/시프트 | 내부 공변량 이동(Internal Covariate Shift) 완화, 더 큰 학습률 허용, 수렴 속도 향상 |
| **ReLU** | 음수 입력은 0으로, 양수는 그대로 통과시키는 비선형 활성함수 | 기울기 소실 문제 완화, 계산 효율 우수, CNN 표준 활성함수 |
| **MaxPool2d(2×2)** | 2×2 영역에서 최댓값만 취함 → 공간 크기 절반 감소 | 평행이동 불변성 부여, 파라미터 없이 feature map 압축, 계산량 감소 |
| **Flatten** | (B, C, H, W) 텐서 → (B, C×H×W) 벡터로 변환 | 완전연결층 입력 형태로 맞춤 |
| **Linear(in → 256)** | 행렬 곱으로 고차원 특징 벡터를 저차원으로 압축 | 추출된 공간 특징을 클래스 결정 경계에 맞게 매핑 |
| **Dropout(p=0.3)** | 학습 중 30% 뉴런을 무작위 비활성화 | 특정 뉴런에 대한 과의존 방지, 앙상블 효과로 과적합 억제 |
| **Linear(256 → 10)** | 최종 분류 점수(로짓) 10개 출력 | CrossEntropyLoss가 소프트맥스를 내부적으로 처리하므로 활성함수 미적용 |

### 3-4. 가중치 초기화

```python
def _init_weights(self):
    for m in self.modules():
        if isinstance(m, nn.Conv2d):
            nn.init.kaiming_normal_(m.weight, nonlinearity="relu")
        elif isinstance(m, nn.Linear):
            nn.init.kaiming_normal_(m.weight, nonlinearity="relu")
```

He(Kaiming) 초기화를 적용하였다. ReLU 활성함수 사용 시 Xavier 초기화보다 He 초기화가 적합하다는 것이 이론적·실험적으로 검증되어 있다(He et al., 2015). ReLU가 입력의 절반을 0으로 만들기 때문에, 신호 분산을 유지하려면 가중치 분산이 2/n_in이 되어야 한다.

### 3-5. Baseline 파라미터 수 계산

| 레이어 | 파라미터 수 계산 | 합계 |
|---|---|---|
| Conv1 (3→32, k=3) | 3×32×3×3 + 32 (bias) | 896 |
| BatchNorm1 | 32×2 (γ, β) | 64 |
| Conv2 (32→64, k=3) | 32×64×3×3 + 64 | 18,496 |
| BatchNorm2 | 64×2 | 128 |
| Conv3 (64→128, k=3) | 64×128×3×3 + 128 | 73,856 |
| BatchNorm3 | 128×2 | 256 |
| FC1 (2048→256) | 2048×256 + 256 | 524,544 |
| FC2 (256→10) | 256×10 + 10 | 2,570 |
| **합계** | | **620,810** |

Flatten 입력 크기: 128채널 × 4 × 4 = 2,048 (32px → 3회 MaxPool → 4px)

### 3-6. Forward Pass

```python
def forward(self, x: torch.Tensor) -> torch.Tensor:
    return self.classifier(self.features(x))
```

`self.features`(합성곱 블록들)가 공간 특징을 추출하고, `self.classifier`(FC 레이어들)가 이를 클래스 로짓으로 변환한다.

## 4. 학습 조건 및 과정 (Part B)

### 4-1. 학습 하이퍼파라미터

| 항목 | 기본값 | 설명 |
|---|---|---|
| **Loss function** | `CrossEntropyLoss` | 다중 분류 표준 손실. 내부적으로 softmax + NLL loss |
| **Optimizer** | `Adam (lr=1e-3)` | 적응적 학습률, 튜닝 부담 적음 |
| **Weight decay** | `5e-4` | L2 정규화 — 가중치 크기 제한으로 과적합 억제 |
| **Epoch** | 70 | 전체 훈련 데이터를 70회 반복 학습 |
| **Batch size** | 128 | 미니배치 크기 |
| **Learning rate** | `1e-3` (Adam), `0.1` (SGD+cosine) | 최적화기에 따라 다름 |

### 4-2. 손실 함수 — Cross-Entropy Loss

$$\mathcal{L} = -\sum_{i=1}^{N} \log \frac{e^{z_{y_i}}}{\sum_{j=1}^{C} e^{z_j}}$$

여기서 $z_j$는 클래스 $j$에 대한 모델의 로짓(raw score), $y_i$는 정답 레이블, $C$는 클래스 수이다. CrossEntropyLoss는 소프트맥스와 NLL(Negative Log-Likelihood)을 수치적으로 안정하게 한 번에 계산한다.

### 4-3. 학습 루프 (1 epoch)

```python
for x, y in train_loader:
    x, y = x.to(device), y.to(device)
    optimizer.zero_grad()         # ① 이전 gradient 초기화
    out = model(x)                 # ② 순전파
    loss = criterion(out, y)       # ③ 손실 계산
    loss.backward()                # ④ 역전파 (gradient 계산)
    optimizer.step()               # ⑤ 가중치 업데이트
```

## 5. 구조 비교 실험 (Part C)

총 5가지 모델 구조를 설계하고 실험하였다. 아래 표에서 baseline 대비 변경된 부분을 굵게 표시하였다.

### 5-1. 실험 모델 명세

| 모델명 | 블록 필터 구성 | 커널 크기 | 블록당 Conv 수 | 풀링 | 파라미터 수 | 변경 포인트 |
|---|---|---|---|---|---|---|
| `baseline` | 32 → 64 → 128 | 3×3 | 1 | 모두 | 620,810 | 기준 모델 |
| `more_filters` | **64 → 128 → 256** | 3×3 | 1 | 모두 | 1,423,114 | **필터 수 2배** |
| `larger_kernel` | 32 → 64 → 128 | **5×5** | 1 | 모두 | 786,186 | **커널 크기 확대** |
| `double_conv` | 32 → 64 → 128 | 3×3 | **2** | 모두 | 815,018 | **블록당 Conv 2회 (VGG-style)** |
| `tuned` | 32 → 64 → 128 | 3×3 | **2** | 모두 | 815,018 | double_conv + **SGD+cosine** |

### 5-2. 모델별 구조 상세

#### baseline
```
[Block 1] Conv(3→32,k=3) → BN → ReLU → MaxPool   → (32, 16, 16)
[Block 2] Conv(32→64,k=3) → BN → ReLU → MaxPool  → (64,  8,  8)
[Block 3] Conv(64→128,k=3) → BN → ReLU → MaxPool → (128, 4,  4)
[FC]      Linear(2048→256) → ReLU → Drop(0.3) → Linear(256→10)
```

#### more_filters — 필터 수 2배 증가
```
[Block 1] Conv(3→64,k=3) → BN → ReLU → MaxPool    → (64,  16, 16)
[Block 2] Conv(64→128,k=3) → BN → ReLU → MaxPool  → (128,  8,  8)
[Block 3] Conv(128→256,k=3) → BN → ReLU → MaxPool → (256,  4,  4)
[FC]      Linear(4096→256) → ReLU → Drop(0.3) → Linear(256→10)
```
필터 수를 2배로 늘리면 각 레이어가 더 많은 특징 패턴을 동시에 학습할 수 있다. 그러나 파라미터 수가 1.4M으로 약 2.3배 증가하여 학습 속도가 느려지고 과적합 위험이 높아질 수 있다.

#### larger_kernel — 5×5 커널 사용
```
[Block 1] Conv(3→32,k=5,pad=2) → BN → ReLU → MaxPool   → (32, 16, 16)
[Block 2] Conv(32→64,k=5,pad=2) → BN → ReLU → MaxPool  → (64,  8,  8)
[Block 3] Conv(64→128,k=5,pad=2) → BN → ReLU → MaxPool → (128, 4,  4)
[FC]      Linear(2048→256) → ReLU → Drop(0.3) → Linear(256→10)
```
5×5 커널은 3×3보다 넓은 수용 영역(Receptive Field)을 한 번에 볼 수 있다. 단, 3×3 커널 2개를 쌓아도 5×5와 동일한 수용 영역을 가지므로(double_conv 아이디어), 반드시 유리하다고는 할 수 없다.

#### double_conv — 블록당 Conv 2회 (VGG-style)
```
[Block 1] Conv(3→32) → BN → ReLU → Conv(32→32) → BN → ReLU → MaxPool  → (32, 16, 16)
[Block 2] Conv(32→64) → BN → ReLU → Conv(64→64) → BN → ReLU → MaxPool → (64,  8,  8)
[Block 3] Conv(64→128)→ BN → ReLU →Conv(128→128)→ BN → ReLU → MaxPool → (128, 4,  4)
[FC]      Linear(2048→256) → ReLU → Drop(0.3) → Linear(256→10)
```
VGGNet에서 제안한 방식으로, 동일한 공간 크기를 유지하면서 비선형 변환을 2회 적용한다. 같은 수용 영역(5×5와 동등)을 더 적은 파라미터로, 더 깊은 비선형성으로 표현한다.

#### tuned — double_conv 구조 + SGD+Cosine 학습법 개선
```
구조: double_conv와 동일
Optimizer: SGD(lr=0.1, momentum=0.9, nesterov=True, weight_decay=5e-4)
Scheduler: CosineAnnealingLR(T_max=70)
```
CIFAR-10의 고전적인 훈련 레시피(SGD+momentum+cosine schedule)를 적용하였다. Adam보다 초기 학습률이 높지만 Cosine Annealing으로 부드럽게 감소시켜 더 좋은 최저점(minimum)에 수렴하는 경향이 있다. `T_max`는 epoch 수와 동일하게 설정되어, 사이클이 훈련 종료와 함께 끝난다.

### 5-3. optimizer 및 scheduler 설정

```python
# Adam (baseline, more_filters, larger_kernel, double_conv)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3, weight_decay=5e-4)

# SGD + Nesterov momentum (tuned)
optimizer = torch.optim.SGD(model.parameters(), lr=0.1, momentum=0.9,
                             nesterov=True, weight_decay=5e-4)

# Cosine Annealing LR Schedule (tuned) — T_max=epochs로 자동 연동
scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=epochs)
```

tuned의 학습률 변화: ep1=0.1000 → ep35=0.0500 → ep55=0.0123 → ep65=0.0018 → ep70=0.0001 (사이클 종료)

## 7. 실험 결과 해석 (Part E)

### 7-1. 어떤 구조가 가장 성능이 좋았는가

최고 성능은 `tuned` 모델로 70 epoch에서 **90.75%** 를 달성하였다.

**구조 개선만의 효과 (double_conv vs baseline, 70 epoch)**

```
baseline:    83.01%  (620K params, Adam, 70ep)
double_conv: 87.57%  (815K params, Adam, 70ep)  → +4.56%p
```

**학습법 개선의 추가 효과 (tuned vs double_conv, 70 epoch)**

```
double_conv: 87.57%  (Adam, lr=1e-3, no scheduler)
tuned:       90.75%  (SGD+momentum, lr=0.1, cosine schedule) → +3.18%p
```

70 epoch에서는 같은 구조임에도 최적화기+스케줄 차이가 3.18%p의 추가 향상을 가져왔다. 50 epoch(+1.15%p) 대비 격차가 더 커진 이유는, Cosine Annealing이 후반부 fine-tuning 구간에서 더 큰 효과를 발휘하기 때문이다.

### 7-2. Epoch 수 증가에 따른 모델별 반응 차이

70 epoch을 50 epoch과 비교하면 모델마다 반응이 뚜렷이 달랐다.

| 모델 | 50ep | 70ep | 변화 | 판정 |
|---|---|---|---|---|
| baseline | 81.85% | 83.01% | +1.16%p | 점진적 개선 |
| more_filters | 84.48% | 85.71% | +1.23%p | 점진적 개선 |
| larger_kernel | 82.80% | 83.24% | +0.44%p | **수렴·과적합 조짐** |
| double_conv | 87.64% | 87.57% | -0.07%p | **plateau, 사실상 수렴** |
| **tuned** | **88.79%** | **90.75%** | **+1.96%p** | **지속 개선** |

**larger_kernel의 과적합**: ep55 이후 val_loss가 0.4699 → 0.4936으로 반등하며 train-val loss 격차가 +0.12 이상으로 벌어졌다. 이는 5×5 커널이 32×32 소형 이미지의 지역 패턴을 지나치게 memorize하기 시작했음을 시사한다.

**tuned의 지속 개선**: ep70에서도 val_loss=0.3040으로 여전히 감소 추세였다. Cosine Annealing이 학습률을 0.1→0.0001로 점진적으로 낮추면서, 후반 epoch일수록 더 정밀한 가중치 조정이 가능하다. 반면 Adam 고정 lr 모델들은 동일한 학습률(1e-3)이 계속 적용되어 후반에 수렴하거나 진동한다.

### 7-3. 각 구조 변경의 효과와 이유

**필터 수 2배 증가 (more_filters: +2.70%p)**

필터 수를 늘리면 각 레이어가 더 다양한 특징 패턴을 동시에 학습한다. 그러나 파라미터 수가 2.3배(620K→1.4M) 증가함에도 불구하고 성능 향상은 상대적으로 작았다. 이는 CIFAR-10의 32×32 소형 이미지에서는 필터 다양성(너비, width) 자체보다 깊이(비선형 변환 횟수, depth)가 더 중요함을 시사한다.

**커널 크기 확대 (larger_kernel: +0.23%p)**

5×5 커널은 한 번에 25픽셀의 수용 영역을 가지므로 3×3(9픽셀)보다 넓은 범위의 패턴을 한 번에 볼 수 있다. 그러나 실제 성능 향상은 70 epoch 기준 +0.23%p에 불과했으며, 과적합까지 발생하였다. 이는 동일한 수용 영역을 3×3 커널 2개로 쌓을 때(double_conv)와 비교하면, 후자가 동일한 수용 영역을 더 적은 파라미터와 더 깊은 비선형성으로 표현하기 때문이다(VGGNet의 핵심 통찰).

**블록당 Conv 2회 (double_conv: +4.56%p)**

가장 큰 구조적 개선을 보였다. VGG-style로 각 블록에 합성곱을 2번 쌓으면:
1. 동일한 공간 해상도에서 비선형 변환을 2배 더 적용 → 더 복잡한 특징 표현 가능
2. 5×5 단일 커널 대비 파라미터 효율 우수 (815K vs 786K로 유사한데 성능은 큰 차이)
3. 각 MaxPool 전에 더 풍부한 표현을 학습하고 압축

**SGD+Cosine (tuned: 구조 동일 대비 +3.18%p)**

Adam은 적응적 학습률로 초반 수렴이 빠르지만, SGD+momentum은 더 낮고 평평한 손실 지형의 최저점을 찾는 경향이 있다. CosineAnnealing으로 학습률을 처음에 높게(0.1) 시작해 부드럽게 줄이면, 탐색(exploration)과 수렴(exploitation)이 자연스럽게 분리된다. 이 효과는 epoch 수가 많을수록 두드러진다.

### 7-4. 모델이 잘못 분류한 사례의 특징

오분류 패턴을 분석하면 세 가지 유형으로 구분된다.

**1. 시각적 유사성으로 인한 혼동**

- `cat ↔ dog`: 털, 귀, 네 발 등 형태적 공통점이 많아 두 클래스 모두 낮은 정확도(cat 79.1%, dog 85.7%)
- `deer ↔ horse`: 네 발 달린 동물로 몸통 형태 유사, 크기 비슷

**2. 배경/색상으로 인한 혼동**

- `airplane ↔ bird`: 하늘 배경의 날개 달린 객체 공유
- `ship ↔ airplane`: 파란 배경(하늘/바다)의 유사성

**3. 이미지 품질 및 구도 문제**

32×32의 저해상도에서는 작거나 비전형적 각도로 촬영된 객체의 경우 사람도 분류하기 어려운 경우가 있다.

### 7-5. CNN 구조 변경과 성능 차이 종합

```
baseline(83.01%, 70ep)
  ↓ 필터 수 증가  → more_filters(85.71%)   +2.70%p  : 표현 너비 확대 효과 있으나 제한적
  ↓ 커널 확대     → larger_kernel(83.24%)  +0.23%p  : 과적합 발생, 큰 커널 효과 미미
```

### 8-6. 10클래스 vs 5클래스 비교 분석

동일한 **tuned 모델(double_conv + SGD+cosine)** 로 10클래스(70ep) 와 5클래스(70ep) 를 비교하여, 순수한 "클래스 수 감소 효과"를 분석하였다.

| 클래스 | 10클래스 tuned | 5클래스 tuned | 차이 |
|---|---|---|---|
| airplane | 91.4% | **94.2%** | **+2.8%p** |
| automobile | 96.0% | **98.8%** | **+2.8%p** |
| bird | 88.2% | **89.8%** | **+1.6%p** |
| cat | 79.1% | **82.7%** | **+3.6%p** |
| dog | 85.7% | **86.5%** | **+0.8%p** |
| 5클래스 평균 | 88.1% | **90.4%** | **+2.3%p** |

**관찰된 패턴:**

모든 클래스에서 정확도가 향상되었다. 전체 평균 기준 +2.3%p 의 향상 폭이다. 이는 클래스 수 감소가 전반적으로 분류를 쉽게 만든다는 직관과 일치한다.

- **cat (+3.6%p)**: 10클래스에서 deer, horse 등 다른 네 발 동물과도 혼동되던 부담이 사라져 가장 큰 개선
- **airplane / automobile (+2.8%p)**: 다른 운송수단(ship, truck)이 없어져 분류 경계가 단순해짐
- **dog (+0.8%p)**: 가장 개선 폭이 작음. 5클래스에서도 cat과의 혼동이 여전히 남아있어, 이 쌍은 본질적으로 어려운 문제임을 시사
- **bird (+1.6%p)**: airplane이 유일한 "비행 객체"로 남아 혼동이 집중되었지만, tuned 모델의 표현력이 충분해 여전히 개선됨

**이전 baseline 기반 비교 대비 달라진 점**: 10클래스 tuned와 5클래스 baseline을 비교했을 때는 bird(-8.7%p), dog(-9.0%p)가 오히려 하락하는 것처럼 보였으나, 이는 모델 간 성능 차이가 섞인 결과였다. 동일 모델(tuned)로 비교하니 모든 클래스가 개선됨을 확인하여, 순수 클래스 수 효과는 양의 방향임이 명확해졌다.

### 8-7. 배운 점

1. **클래스 수를 줄이면 전반적으로 분류 난이도가 낮아진다**: 5클래스 tuned는 10클래스 tuned 대비 모든 클래스에서 정확도가 향상되었으며, 평균 +2.3%p의 개선이 있었다.

2. **향상 폭은 클래스별로 크게 다르다**: 경쟁 클래스가 많이 제거된 cat(+3.6%p), airplane/automobile(+2.8%p)의 향상 폭이 컸던 반면, 본질적으로 유사한 쌍(cat-dog)이 남은 dog(+0.8%p)는 개선 폭이 가장 작았다.

3. **비교 실험에서는 조건 통제가 중요하다**: 초기 분석에서 subset_baseline을 10클래스 tuned와 비교했을 때 bird/dog 정확도가 하락한 것처럼 보였지만, 이는 **모델 차이**와 **클래스 수 차이**가 혼재된 잘못된 비교였다. 동일 모델(tuned)로 통제한 결과 순수 클래스 수 효과는 일관된 양의 방향으로 확인되었다.

4. **"어려운" 클래스 쌍(cat-dog)은 구조적 한계를 가진다**: 클래스 수를 줄이고 학습법을 개선해도 dog의 향상 폭이 가장 작았던 점은, 이 쌍의 혼동이 모델 용량이나 학습 전략 개선만으로 완전히 해결되기 어려움을 시사한다.

## 9. 추가 기법 적용 (Part G)

### 9-1. Dropout

**적용 위치**: FC 레이어 사이 (`Linear(2048→256) → ReLU → Dropout(p=0.3) → Linear(256→10)`)

**원리**: 학습 중 매 forward pass마다 30%의 뉴런을 무작위로 비활성화한다. 이는 각 뉴런이 다른 뉴런에 과도하게 의존하지 않도록 강제하여, 여러 독립적인 서브네트워크들의 앙상블 효과를 낸다. 추론 시에는 모든 뉴런을 활성화하고 출력을 (1-p)로 스케일링한다.

```python
self.classifier = nn.Sequential(
    nn.Flatten(),
    nn.Linear(flat_dim, fc_dim),
    nn.ReLU(inplace=True),
    nn.Dropout(dropout),          # p=0.3
    nn.Linear(fc_dim, num_classes),
)
```

**효과**: Dropout 없이 같은 조건으로 훈련하면 train accuracy >> val accuracy 형태의 과적합이 쉽게 발생한다. Dropout은 val_loss를 낮추고 train-val 간 격차를 줄이는 정규화 효과를 제공한다. 이번 실험에서 tuned 70 epoch 기준 train_acc 95.7% vs val_acc 90.75%의 격차(약 5%p)는 Dropout이 없었다면 더 커졌을 것이다.

### 9-2. Data Augmentation

```python
train_tf = transforms.Compose([
    transforms.RandomCrop(32, padding=4),
    transforms.RandomHorizontalFlip(),
    transforms.ToTensor(),
    transforms.Normalize(MEAN, STD),
])
```

**RandomCrop(32, padding=4)**: 이미지 가장자리에 4픽셀 zero-padding을 추가한 뒤(36×36), 무작위 위치에서 32×32를 잘라낸다. 이는 객체의 위치가 이미지 내에서 약간 이동해도 분류가 안정적이도록 한다.

**RandomHorizontalFlip()**: 50% 확률로 이미지를 좌우 반전한다. 대부분의 객체는 방향에 관계없이 같은 클래스이므로(자동차는 왼쪽을 봐도, 오른쪽을 봐도 자동차) 유효한 augmentation이다.

**효과**: 50,000장의 훈련 데이터를 사실상 무한히 변형된 버전으로 학습함으로써, 위치·방향에 불변한 특징을 학습하게 한다. 특히 후반 epoch에서 augmentation이 없다면 훈련 이미지가 반복 노출되어 과적합이 가속된다.

### 9-3. Feature Map 시각화

```python
@torch.no_grad()
def feature_maps(self, x: torch.Tensor) -> List[torch.Tensor]:
    outs: List[torch.Tensor] = []
    for layer in self.features:
        x = layer(x)
        if isinstance(layer, nn.ReLU):
            outs.append(x.detach().cpu())
    return outs
```

각 ReLU 레이어 직후의 활성화 맵을 시각화한다.

![Feature Map 시각화 — tuned 모델](runs/full_20260421_213313/feature_maps_tuned.png)

| 층 | 관찰되는 특징 | 해석 |
|---|---|---|
| Block 1 (ReLU 1) | 엣지, 색상 경계, 방향성 패턴 | 저수준 특징 — Gabor 필터와 유사 |
| Block 1 (ReLU 2) | 더 정제된 엣지, 지역 텍스처 | 두 번째 Conv가 첫 번째 결과를 조합 |
| Block 2 (ReLU 3) | 코너, 곡선, 색상 블롭 | 중수준 특징 — 부분 형태 |
| Block 2 (ReLU 4) | 물체 부분(다리, 날개 등) | 더 추상적인 패턴 |
| Block 3 (ReLU 5~6) | 추상적, 시각적으로 해석 어려움 | 고수준 의미적 특징 |

초기 층에서 후기 층으로 갈수록 feature map이 점점 더 추상적이 되는 것은 계층적 표현 학습(Hierarchical Feature Learning)의 전형적인 모습이다.
