## S2. 문제 정의 — 왜 멀티모달이 필요한가 (50초)

**제목**: 단일 데이터로는 부족하다
**3카드 구성**:
- ① 센서만 보면? — 진동·온도·각도는 알지만 시각적 결함(부품 빠짐 등)은 못 봄
- ② 이미지만 보면? — 시각적 결함은 보이지만 작동 상태(부하·속도)는 알 수 없음
- ③ 두 데이터를 함께 쓰면? — 서로의 한계를 보완할 수 있을 것이라는 기대

**하단 강조**: → 그런데 정말 결합이 항상 더 좋을까? 어떤 조건에서 효과가 있을까?

**스크립트** (약 50초):
> 제조 라인의 자동 검수 시스템은 보통 두 종류의 데이터를 활용합니다. 첫 번째는 로봇의 부하나 관절 각도, 컨베이어 온도 같은 센서 측정값이고, 두 번째는 카메라로 찍은 제품 이미지입니다.
> 그런데 이 두 가지를 따로따로 쓰면 각자 놓치는 부분이 생깁니다. 센서값만 보면 부품이 시각적으로 빠진 결함을 알기 어렵고, 이미지만 보면 로봇이 비정상적인 힘으로 작동하는 상황은 알 수 없습니다.
> 그래서 두 모달리티를 결합하는 멀티모달 접근이 필요한데, 핵심 질문은 이겁니다. 정말 결합이 항상 더 좋은가, 그렇다면 어떤 조건에서 효과가 있는가. 저희 프로젝트는 이 질문에 실험으로 답하는 것이 목표였습니다.

## S4. 데이터셋 소개 (60초)

**제목**: Future Factories Dataset (사우스캐롤라이나대)
**좌측 텍스트**:
- 로봇 4대 + 컨베이어로 구성된 실제 조립 라인
- 4개 부품으로 된 로켓 모형 조립
- 30시간 가동, **16만 6천 건의 측정 데이터** 수집

**우측 표 (이상 유형 분포)**:
| 라벨 | 비율 |
|------|------|
| Normal (정상) | 54.7% |
| NoNose | 11.6% |
| NoNose + NoBody2 | 15.2% |
| NoNose + NoBody2 + NoBody1 | 16.0% |
| 기타 | 2.4% |

**하단 강조 박스** (★ 핵심 — 청중 이해 도움):
- **22채널 센서값**: 모든 측정 순간에 기록됨 (그리퍼 부하, 관절 각도, 온도 등)
- **카메라 2대 동기 촬영**: 부품이 명확히 보이는 특정 단계(Cycle State 4·9)에서 사용
- **라벨**: 매 측정 순간마다 그 시점의 상태(정상/어떤 부품 빠짐)가 기록되어 있음

**스크립트** (약 60초):
> 저희가 사용한 데이터는 사우스캐롤라이나 대학 Future Factories 연구실에서 공개한 산업용 데이터셋입니다. 실제 조립 라인을 그대로 옮긴 시설에서 로봇 4대와 컨베이어가 30시간 동안 로켓 모형을 만들었고, 그동안 16만 6천 건의 측정 데이터가 쌓였습니다.
> 측정 데이터에는 두 종류가 들어 있습니다. 22개 센서값은 모든 측정 순간에 기록되었고, 카메라 두 대가 찍은 이미지는 부품이 명확히 보이는 특정 공정 단계에서 사용했습니다.
> 결함은 의도적으로 만들었습니다. 부품을 빼고 조립하게 해서 정상 외에 NoNose, NoBody1 같은 결함 유형이 만들어졌고, 매 측정 순간마다 그 시점에 어떤 상태였는지가 라벨로 기록되어 있습니다. 즉 저희 문제는 "지금 이 순간 어떤 상태인가"를 매 시점마다 분류하는 과제입니다.

## S5. 모델 설계 — 세 가지 모델 (60초)

**제목**: 세 가지 모델로 비교 실험
**3컬럼**:
- **센서 단독 (BiLSTM)**
  - 입력: 22채널 시계열
  - 양방향 LSTM으로 시간 흐름 분석
  - 역할: 모든 시점에서 작동하는 기본 모델

- **이미지 단독 (ResNet18)**
  - 입력: 카메라 2대 이미지
  - ImageNet 사전학습 모델 미세조정
  - 두 카메라의 특징을 평균
  - 역할: 시각적 결함 직접 판별

- **융합 모델 (Decision-level)**
  - 두 모델의 출력 확률을 가중 결합
  - 이미지가 없으면 자동으로 센서만 사용
  - 역할: 두 모달리티의 정보를 종합

**하단 강조**: → 세 모델 모두 동일한 학습/테스트 데이터로 학습하여 직접 비교 가능

**스크립트** (약 60초):
> 세 가지 모델을 만들어 비교했습니다.
> 첫 번째는 센서 단독 모델로, 양방향 LSTM을 사용해 22채널 센서값의 시간적 흐름을 분석합니다. 이 모델은 센서가 항상 측정되기 때문에 모든 시점에서 작동합니다.
> 두 번째는 이미지 단독 모델로, ImageNet으로 사전학습된 ResNet18을 사용합니다. 카메라가 두 대이기 때문에 두 이미지를 각각 통과시킨 뒤 특징을 평균내는 방식으로 두 시점의 정보를 모두 활용했습니다.
> 세 번째는 융합 모델인데, 두 모델의 예측 확률을 가중 결합하는 Decision-level 방식을 채택했습니다. 이 방식의 장점은 이미지가 들어오지 않는 순간에는 자동으로 센서 모델만 사용하도록 설계할 수 있어서, 결측 상황에서도 안정적으로 작동한다는 점입니다.

## S6. 시스템 전체 구조 (40초)

**제목**: 전체 파이프라인
**다이어그램** (좌→우 흐름):
```
[센서 데이터 (모든 시점)] ──→ [BiLSTM] ──→ [센서 예측 확률]
                                                    │
                                                    ▼
[카메라 이미지 (해당 시점)] ──→ [ResNet18 x2] ──→ [이미지 예측 확률] ──→ [확률 가중 결합] ──→ [최종 분류: 정상/결함 유형]
                                                                              ▲
                                                            (이미지 없으면 센서만 사용)
```

**스크립트** (약 40초):
> 전체 시스템은 이렇게 구성됩니다.
> 센서값은 항상 들어오기 때문에 매 시점마다 BiLSTM이 예측 확률을 만듭니다. 카메라 이미지는 부품이 보이는 시점에 들어오는데, 두 대의 이미지를 각각 ResNet18에 통과시켜 특징을 합친 뒤 이미지 예측 확률을 만듭니다.
> 마지막에 두 확률을 가중 평균해서 최종 분류 결과를 냅니다. 만약 이미지가 들어오지 않는 시점이면 자동으로 센서 확률만 사용하기 때문에, 시스템은 어떤 상황에서도 멈추지 않고 작동합니다.

## S7. 핵심 실험 결과 (75초)

**제목**: 융합이 단독보다 나은가
**바 그래프 (res_present.png)**: Sensor 0.974 / Image 0.928 / Fusion-Concat 0.981 / Fusion-CrossAttn 0.980 / Decision-Fusion 0.928

**평가 조건 명시** (그래프 위):
> 평가 대상: **카메라 이미지가 함께 입력되는 시점 2,161개에서의 분류 성능 (Weighted F1)**

**핵심 발견** (하단):
- 센서 단독: **0.974** (이 시점에서 이미 매우 높은 성능)
- 이미지 단독: **0.928** (두 카메라 결합 효과)
- **융합 모델: 0.981** → 두 단독 모델을 모두 능가
- 비록 차이는 작지만, **융합이 두 정보를 종합해 더 정확한 판단을 내릴 수 있음을 확인**

**스크립트** (약 75초):
> 첫 번째 핵심 결과입니다. 이 그래프는 카메라 이미지가 함께 입력되는 시점, 즉 두 모달리티 데이터가 모두 있는 2,161개 시점에서 각 모델의 성능을 측정한 것입니다. 평가 지표는 클래스 불균형을 고려한 Weighted F1 점수입니다.
> 결과를 보시면, 센서 단독 모델이 0.974, 카메라 두 대를 결합한 이미지 단독 모델이 0.928, 그리고 두 모델을 결합한 융합 모델이 0.981로 가장 높습니다.
> 차이가 크지는 않지만 중요한 점은 융합 모델이 센서·이미지 어느 한쪽보다도 더 나은 성능을 보였다는 것입니다. 이는 두 모달리티가 서로 다른 정보를 제공하고 있고, 융합 모델이 그 정보를 종합해서 더 정확한 판단을 내릴 수 있다는 것을 의미합니다. 즉 멀티모달 결합의 효과가 실제로 존재한다는 것을 보여줍니다.

## S8. 추가 검증 — 모달리티 균형 (75초)

**제목**: 융합 효과는 언제 발생하는가
**좌측 설명**:
- 의문: 만약 이미지가 너무 강하다면? 또는 너무 약하다면?
- 답을 얻기 위한 **추가 실험**:
- 이미지 모델의 학습량을 인위적으로 조절하여 강도를 바꾸고, 각 경우에서 융합 효과를 측정

**우측 결과 그래프** (3시나리오 막대):
| 시나리오 | 센서 F1 | 이미지 F1 | 융합 F1 | 결과 |
|---|---|---|---|---|
| 이미지 약함 (3에폭 학습) | 0.974 | 0.775 | 0.978 | 융합 우위 |
| 두 모달 약화 (균형) | 0.956 | 0.775 | 0.968 | 융합 우위 |
| 이미지 강함 (충분 학습) | 0.974 | 0.928 | 0.981 | 융합 우위 |

**핵심 결론** (하단):
- **세 가지 강도 시나리오 모두에서 융합이 단독을 능가**
- 멀티모달 결합의 효과는 **두 모달리티가 동시에 가용한 상황에서 일관되게 나타남**

**스크립트** (약 75초):
> 그런데 한 가지 의문이 들 수 있습니다. 만약 두 모달리티 중 한쪽이 너무 강하다면 다른 쪽을 결합해도 의미가 없는 것 아닌가, 반대로 한쪽이 너무 약하면 오히려 융합 모델이 끌려 내려가는 것 아닌가 하는 의문입니다.
> 이걸 확인하기 위해 추가 실험을 했습니다. 이미지 모델의 학습량을 인위적으로 조절해서 강한 이미지·약한 이미지·중간 강도 이렇게 세 가지 시나리오를 만들고, 각 경우에서 융합 성능을 측정했습니다.
> 결과를 보시면, 이미지가 약한 경우, 두 모달이 비등한 경우, 이미지가 강한 경우 모두에서 융합 모델이 두 단독 모델을 능가했습니다. 즉 두 모달리티의 강도가 다양하게 변해도 융합 효과가 일관되게 나타난다는 것이 확인되었습니다. 이는 멀티모달 접근이 특정 조건에서만 운 좋게 작동하는 것이 아니라, 일반적으로 유효한 전략임을 보여줍니다.

## S9. 결함 유형별 분석 (50초)

**제목**: 어떤 결함이 잘 잡히고, 어떤 결함이 어려운가
**그래프 (res_byclass.png)**: NoNose 0.98 / NoNose_NoBody2 0.87 / NoNose_NoBody2_NoBody1 0.59 / Normal 0.92

**해석**:
- **단순 결함 (NoNose)** — F1 0.98로 거의 완벽하게 검출
- **복합 결함 2개 부품 (NoNose + NoBody2)** — F1 0.87로 양호
- **복합 결함 3개 부품 (NoNose + NoBody2 + NoBody1)** — F1 0.59로 가장 어려움
- 부품이 여러 개 빠진 복합 결함일수록 결함끼리 시각적·센서적으로 유사해져 구분이 어려움

**스크립트** (약 50초):
> 결함 유형별로 모델이 얼마나 잘 작동하는지 분석한 결과입니다.
> NoNose 같은 단순한 단일 부품 결함은 F1 0.98로 거의 완벽하게 검출됩니다. 두 부품이 빠진 복합 결함은 0.87로 양호한 편입니다. 그런데 세 부품이 동시에 빠진 가장 복잡한 결함은 0.59로 검출 성능이 크게 떨어집니다.

## S10. 결론 (60초)

**프로젝트 의의**:
- 단순히 "멀티모달이 좋다"가 아니라, "어떤 조건에서 어떻게 좋은지"를 정량적으로 규명

> 둘째, 이미지 모델의 강도를 다양하게 바꿔서 추가 실험을 한 결과, 어떤 강도 조건에서도 융합 모델이 단독 모델을 능가해 융합 효과의 일관성을 확인했습니다.
> 셋째, 이미지가 들어오지 않는 시점에는 자동으로 센서 모델만 사용하도록 설계해서, 결측 상황에서도 시스템이 안정적으로 작동합니다.
> 즉 저희 프로젝트의 의의는 단순히 멀티모달이 좋다는 결론을 내리는 것이 아니라, 어떤 조건에서 어떻게 좋은지를 실험으로 밝히고, 실제 산업 환경에 적용 가능한 견고한 구조를 설계했다는 점입니다.
