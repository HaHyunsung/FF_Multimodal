## S2. 문제 정의 — 왜 멀티모달이 필요한가 (50초)

**제목**: 단일 데이터로는 부족하다
**3카드 구성**:
- ① 센서만 보면? — 진동·온도·각도는 알지만 시각적 결함(부품 빠짐 등)은 못 봄
- ② 이미지만 보면? — 시각적 결함은 보이지만 작동 상태(부하·속도)는 알 수 없음
- ③ 두 데이터를 함께 쓰면? — 서로의 한계를 보완할 수 있을 것이라는 기대

**하단 강조**: → 그런데 정말 결합이 항상 더 좋을까? 어떤 조건에서 효과가 있을까?

**스크립트** (약 50초):
> 제조 라인의 자동 검수 시스템은 보통 두 종류의 데이터를 활용합니다. 첫 번째는 로봇의 부하나 관절 각도, 컨베이어 온도 같은 센서 측정값이고, 두 번째는 카메라로 찍은 제품 이미지입니다.
> 그런데 이 두 가지를 따로따로 쓰면 각자 놓치는 부분이 생깁니다. 센서값만 보면 부품이 시각적으로 빠진 결함을 알기 어렵고, 이미지만 보면 로봇이 비정상적인 힘으로 작동하는 상황은 알 수 없습니다.
> 그래서 두 모달리티를 결합하는 멀티모달 접근이 필요한데, 핵심 질문은 이겁니다. 정말 결합이 항상 더 좋은가, 그렇다면 어떤 조건에서 효과가 있는가. 저희 프로젝트는 이 질문에 실험으로 답하는 것이 목표였습니다.

- ② **언제 이득이 발생하는가?** 두 모달리티의 강도를 조절하며 융합 효과 분석
- ③ **이미지가 없는 순간엔?** 카메라 데이터가 없는 구간에서도 작동하는 견고한 융합 설계

**스크립트** (약 40초):
> 저희 프로젝트는 세 가지 구체적인 질문에 답하려 했습니다.
> 첫째, 멀티모달 결합 모델이 센서 단독, 이미지 단독보다 실제로 더 좋은 성능을 내는가. 둘째, 두 모달리티 중 한쪽이 너무 강하거나 약할 때 융합 효과가 어떻게 달라지는가. 셋째, 카메라 이미지가 항상 들어오는 것이 아닐 때, 즉 이미지가 없는 순간에도 모델이 안정적으로 작동할 수 있는가입니다.
> 이 세 가지 질문에 정량적으로 답하는 것이 저희 프로젝트의 핵심입니다.

## S4. 데이터셋 소개 (60초)

**제목**: Future Factories Dataset (사우스캐롤라이나대)
**좌측 텍스트**:
- 로봇 4대 + 컨베이어로 구성된 실제 조립 라인
- 4개 부품으로 된 로켓 모형 조립
- 30시간 가동, **16만 6천 건의 측정 데이터** 수집

**우측 표 (이상 유형 분포)**:
| 라벨 | 비율 |
|------|------|
| Normal (정상) | 54.7% |
| NoNose | 11.6% |
| NoNose + NoBody2 | 15.2% |
| NoNose + NoBody2 + NoBody1 | 16.0% |
| 기타 | 2.4% |

**하단 강조 박스** (★ 핵심 — 청중 이해 도움):
- **22채널 센서값**: 모든 측정 순간에 기록됨 (그리퍼 부하, 관절 각도, 온도 등)
- **카메라 2대 동기 촬영**: 부품이 명확히 보이는 특정 단계(Cycle State 4·9)에서 사용
- **라벨**: 매 측정 순간마다 그 시점의 상태(정상/어떤 부품 빠짐)가 기록되어 있음

**스크립트** (약 60초):
> 저희가 사용한 데이터는 사우스캐롤라이나 대학 Future Factories 연구실에서 공개한 산업용 데이터셋입니다. 실제 조립 라인을 그대로 옮긴 시설에서 로봇 4대와 컨베이어가 30시간 동안 로켓 모형을 만들었고, 그동안 16만 6천 건의 측정 데이터가 쌓였습니다.
> 측정 데이터에는 두 종류가 들어 있습니다. 22개 센서값은 모든 측정 순간에 기록되었고, 카메라 두 대가 찍은 이미지는 부품이 명확히 보이는 특정 공정 단계에서 사용했습니다.
> 결함은 의도적으로 만들었습니다. 부품을 빼고 조립하게 해서 정상 외에 NoNose, NoBody1 같은 결함 유형이 만들어졌고, 매 측정 순간마다 그 시점에 어떤 상태였는지가 라벨로 기록되어 있습니다. 즉 저희 문제는 "지금 이 순간 어떤 상태인가"를 매 시점마다 분류하는 과제입니다.

## S5. 모델 설계 — 세 가지 모델 (60초)

**제목**: 세 가지 모델로 비교 실험
**3컬럼**:
- **센서 단독 (BiLSTM)**
  - 입력: 22채널 시계열
  - 양방향 LSTM으로 시간 흐름 분석
  - 역할: 모든 시점에서 작동하는 기본 모델

- **이미지 단독 (ResNet18)**
  - 입력: 카메라 2대 이미지
  - ImageNet 사전학습 모델 미세조정
  - 두 카메라의 특징을 평균
  - 역할: 시각적 결함 직접 판별

- **융합 모델 (Decision-level)**
  - 두 모델의 출력 확률을 가중 결합
  - 이미지가 없으면 자동으로 센서만 사용
  - 역할: 두 모달리티의 정보를 종합

**하단 강조**: → 세 모델 모두 동일한 학습/테스트 데이터로 학습하여 직접 비교 가능

**스크립트** (약 60초):
> 세 가지 모델을 만들어 비교했습니다.
> 첫 번째는 센서 단독 모델로, 양방향 LSTM을 사용해 22채널 센서값의 시간적 흐름을 분석합니다. 이 모델은 센서가 항상 측정되기 때문에 모든 시점에서 작동합니다.
> 두 번째는 이미지 단독 모델로, ImageNet으로 사전학습된 ResNet18을 사용합니다. 카메라가 두 대이기 때문에 두 이미지를 각각 통과시킨 뒤 특징을 평균내는 방식으로 두 시점의 정보를 모두 활용했습니다.
> 세 번째는 융합 모델인데, 두 모델의 예측 확률을 가중 결합하는 Decision-level 방식을 채택했습니다. 이 방식의 장점은 이미지가 들어오지 않는 순간에는 자동으로 센서 모델만 사용하도록 설계할 수 있어서, 결측 상황에서도 안정적으로 작동한다는 점입니다.

## S6. 시스템 전체 구조 (40초)

**제목**: 전체 파이프라인
**다이어그램** (좌→우 흐름):
```
[센서 데이터 (모든 시점)] ──→ [BiLSTM] ──→ [센서 예측 확률]
                                                    │
                                                    ▼
[카메라 이미지 (해당 시점)] ──→ [ResNet18 x2] ──→ [이미지 예측 확률] ──→ [확률 가중 결합] ──→ [최종 분류: 정상/결함 유형]
                                                                              ▲
                                                            (이미지 없으면 센서만 사용)
```

**스크립트** (약 40초):
> 전체 시스템은 이렇게 구성됩니다.
> 센서값은 항상 들어오기 때문에 매 시점마다 BiLSTM이 예측 확률을 만듭니다. 카메라 이미지는 부품이 보이는 시점에 들어오는데, 두 대의 이미지를 각각 ResNet18에 통과시켜 특징을 합친 뒤 이미지 예측 확률을 만듭니다.
> 마지막에 두 확률을 가중 평균해서 최종 분류 결과를 냅니다. 만약 이미지가 들어오지 않는 시점이면 자동으로 센서 확률만 사용하기 때문에, 시스템은 어떤 상황에서도 멈추지 않고 작동합니다.

## S7. 중간발표 → 최종: 가설과 검증 (45초) ★신규

**제목**: 중간발표 → 최종: 가설과 검증
**3카드 구성**:
- ① **중간발표: 융합이 졌다** — 전체 구간에서 Fusion F1 0.88 < 센서 0.93 (멀티모달이 단독보다 낮음)
- ② **원인 가설: 데이터 균형** — 이미지 9.5천 장 vs 센서 12만 시퀀스, 불균형으로 이미지가 노이즈로 작용한다고 진단
- ③ **최종: 가설 검증됨** — 이미지 커버리지 확장 + 융합전략 개선 → 균형 구간에서 Fusion 0.986 > 두 단독 (유의)

**하단 강조**: → 멀티모달은 무조건 이기지 않는다. 두 모달이 함께·균형 있게 있을 때 이긴다 — 이 조건을 정량으로 규명

**스크립트** (약 45초):
> 여기서 저희 프로젝트가 어떻게 발전했는지 말씀드리겠습니다. 사실 중간발표 때는 융합 모델이 전체 구간에서 F1 0.88로, 센서 단독 0.93보다 오히려 낮았습니다. 멀티모달인데 단독보다 못한 결과였죠.
> 저희는 그 원인을 데이터 불균형으로 진단했습니다. 이미지는 9천여 장인데 센서는 12만 시퀀스라, 융합에서 이미지가 정보를 더하기보다 노이즈로 작용한 겁니다.
> 그래서 최종 단계에서 이미지 커버리지를 늘리고 융합 방식을 개선했고, 두 모달리티가 균형 있게 함께 들어오는 구간에서는 융합이 두 단독을 모두 넘어섰습니다. 즉 저희가 세운 가설, ‘멀티모달의 성공은 데이터 균형에 달려 있다’를 직접 검증한 것입니다.

## S8. 핵심 실험 결과 (55초)

**제목**: 융합이 단독보다 나은가
**바 그래프 (res_present.png)**: Sensor 0.907 / Image 0.925 / Fusion-Concat 0.921 / Fusion-CrossAttn 0.913 / **Decision-Fusion 0.986**

**평가 조건 명시** (그래프 위):
> 평가 대상: **카메라 이미지가 함께 입력되는 시점 3,912개에서의 분류 성능 (Weighted F1)**

**핵심 발견** (하단):
- 센서 단독: **0.907**
- 이미지 단독: **0.925** (두 카메라 결합 효과)
- **융합 모델(Decision-Fusion): 0.986** → 두 단독 모델을 모두 뚜렷하게 능가
- 부트스트랩 검정으로 **융합 − 센서 = +0.078, 95% 신뢰구간이 0보다 큼 → 통계적으로 유의**

**스크립트** (약 55초):
> 앞서 세운 가설을 검증한 핵심 근거가 이 그래프입니다. 두 모달리티 데이터가 모두 있는 3,912개 시점에서 측정했고, 지표는 클래스 불균형을 고려한 Weighted F1입니다.
> 센서 단독 0.907, 이미지 단독 0.925, 그리고 두 모델의 예측 확률을 가중 결합한 융합 모델이 0.986으로 두 단독을 모두 뚜렷하게 능가했습니다.
> 이 차이는 우연이 아닙니다. 부트스트랩 검정 결과 융합과 센서의 차이가 평균 0.078, 95% 신뢰구간이 0보다 커서 통계적으로 유의했습니다. 두 모달리티가 서로 다른 정보를 담고 있고, 융합이 그것을 종합해 더 정확히 판단한다는 것을 보여줍니다.

## S9. 추가 검증 — 모달리티 균형 (60초)

**제목**: 융합 효과는 언제 발생하는가
**좌측 설명**:
- 의문: 만약 이미지가 너무 강하다면? 또는 너무 약하다면?
- 답을 얻기 위한 **추가 실험**:
- 이미지 모델의 학습량을 인위적으로 조절하여 강도를 바꾸고, 각 경우에서 융합 효과를 측정

**우측 결과 그래프** (2시나리오 막대, 모두 이미지 존재 3,912 시점, 융합=Decision-Fusion):
| 시나리오 | 센서 F1 | 이미지 F1 | 융합 F1 | 결과 |
|---|---|---|---|---|
| 이미지 약함 (덜 학습) | 0.907 | 0.264 | 0.908 | 손해 없음 (센서 수준 유지) |
| 이미지 강함 (충분 학습) | 0.907 | 0.925 | 0.986 | +0.061 큰 이득 |

> ※ 센서까지 함께 약화시킨 '균형 통제' 실험도 했으나, 이미지 존재 구간에선 '이미지 약함'과 거의 동일(센서 0.898·융합 0.900)해 차트에선 제외. 전체 셋 기준 결과는 보고서에 수록.

**핵심 결론** (하단):
- 이미지가 약할 때 융합은 **센서 수준을 안전하게 유지** (약한 정보에 끌려 내려가지 않음)
- 이미지가 강할 때 융합은 **센서보다 크게 향상** → 이득은 이미지 품질에 비례
- 즉 융합은 **손해 없이 작동하며, 정보가 유용할수록 더 좋아짐**

**스크립트** (약 60초):
> 그렇다면 이미지가 항상 좋은 상태로 들어오지 않을 때는 어떨까요? 조명이 나쁘거나 학습이 부족해 이미지가 약할 때, 그걸 결합하면 융합이 오히려 끌려 내려가지 않을까 하는 의문입니다.
> 그래서 이미지 모델을 일부러 덜 학습시켜 약하게 만든 경우와, 충분히 학습시켜 강하게 만든 경우 두 가지를 비교했습니다.

> 즉 융합은 이미지가 도움이 안 되면 센서 수준을 지키고, 유용하면 그만큼 이득을 가져갑니다. 결측과 품질 저하가 흔한 실제 산업 현장에서 융합이 안전한 선택인 이유입니다.

## S11. 결론 (60초)

**제목**: 우리가 확인한 것
**3가지 핵심 결론**:
- ① **멀티모달 융합의 효과를 정량적으로 확인** — 두 모달이 함께 입력되는 시점에서 융합이 센서·이미지 단독 모두를 능가 (F1 0.986 vs 센서 0.907, 이미지 0.925 · 통계적으로 유의)
- ② **융합은 손해 없이 작동** — 이미지가 약하면 센서 수준 유지, 강하면 이득 증가
- ③ **결측 모달리티에 견고한 시스템** — 이미지가 없는 시점에는 자동으로 센서만 사용해 안정적으로 작동

**프로젝트 의의**:
- 단순히 "멀티모달이 좋다"가 아니라, "어떤 조건에서 어떻게 좋은지"를 정량적으로 규명
- 실제 산업 환경(이미지가 항상 가용하지 않은 상황)에 적용 가능한 견고한 구조 설계

**스크립트** (약 60초):
> 결론입니다. 저희 프로젝트는 세 가지를 확인했습니다.
> 첫째, 두 모달리티가 함께 입력되는 시점에서 융합 모델이 센서 단독 0.907, 이미지 단독 0.925보다 높은 F1 0.986을 기록했고, 이 차이는 통계적으로도 유의했습니다. 멀티모달 결합의 효과를 정량적으로 확인한 것입니다.
> 둘째, 이미지의 강도를 약하게도 강하게도 바꿔 본 결과, 이미지가 약할 때 융합은 센서 수준을 안전하게 유지하고 이미지가 강할 때는 그만큼 이득을 가져갔습니다. 즉 융합은 손해 없이 작동하며 정보가 좋을수록 더 좋아집니다.
> 셋째, 이미지가 들어오지 않는 시점에는 자동으로 센서 모델만 사용하도록 설계해서, 결측 상황에서도 시스템이 안정적으로 작동합니다.
> 즉 저희 프로젝트의 의의는 단순히 멀티모달이 좋다는 결론을 내리는 것이 아니라, 어떤 조건에서 어떻게 좋은지를 실험으로 밝히고, 실제 산업 환경에 적용 가능한 견고한 구조를 설계했다는 점입니다.
