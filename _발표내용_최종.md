## S2. 서론: 왜 멀티모달인가

- **단일 모달리티의 한계**: 센서만 / 이미지만으로는 놓치는 이상이 존재
- **공정의 복합성**: 온도·속도·로봇 관절·시각 상태 등 여러 변수가 상호작용
- **멀티모달의 기대**: 한 모달리티가 약한 순간을 다른 모달리티가 보완
- → 핵심 질문: "융합은 항상 이득인가?"

**스크립트**: 제조 라인의 이상 탐지는 품질·효율에 직결됩니다. 그런데 센서값만 보거나 이미지만 보면, 각자 놓치는 지점이 생깁니다. 예를 들어 부품이 미세하게 빠진 결함은 이미지로는 잘 안 보여도 로봇 그리퍼 부하 같은 센서에는 흔적이 남습니다. 그래서 두 모달리티를 합치면 서로를 보완할 것이라 기대했는데, 저희 실험은 "융합이 항상 이득은 아니다"라는 흥미로운 결과로 이어졌습니다.

## S3. 프로젝트 목표

- 동일 데이터셋에서 **센서 단독 / 이미지 단독 / 융합** 3종 모델을 공정하게 비교
- **멀티모달 융합 이득의 성립 조건**을 정량적으로 규명
- 결측 모달리티(이미지가 없는 구간)에 견고한 융합 전략 설계

**스크립트**: 저희 목표는 세 가지입니다. 첫째, 센서 단독, 이미지 단독, 그리고 융합 모델을 같은 잣대로 공정하게 비교하는 것. 둘째, 융합이 이득을 주는 조건이 무엇인지 정량적으로 밝히는 것. 셋째, 이미지가 늘 있는 게 아니기 때문에, 이미지가 없는 구간에서도 무너지지 않는 융합 구조를 만드는 것입니다.

## S4. 데이터셋 (시각자료: sensor_sample.png + 카메라 이미지[갱신] + 표)

- **Future Factories Lab Dataset (USC)**: 로봇 4대 + 컨베이어, 4부품 로켓 조립
- 30시간 가동, **166,000 레코드, 285 사이클**, 22채널 센서 + **카메라 2대**
- 의도적 부품 제거로 결함 생성 → 5클래스 (Normal/NoNose/NoBody1/NoNose_NoBody2/...)
- **카메라는 Cycle State 4·9에서만 촬영** (부품이 보이는 시점) → 이미지는 전체의 일부에만 존재
- 센서 샘플: 정상 vs 이상에서 그리퍼 부하·관절 각도·온도 패턴 차이 (그래프)

**스크립트**: 데이터는 사우스캐롤라이나 대학 Future Factories Lab의 산업용 데이터셋입니다. 로봇 4대와 컨베이어가 로켓 모형을 조립하며 30시간 동안 16만 6천 건의 레코드를 수집했고, 22개 센서 채널과 카메라 2대 이미지가 동기화돼 있습니다. 여기 센서 그래프를 보시면, 그리퍼 부하나 관절 각도가 정상과 이상에서 다른 패턴을 보입니다. 한 가지 중요한 특징은, 카메라가 사이클 안의 4번과 9번 단계에서만 촬영한다는 점입니다. 즉 이미지는 전체 시점의 일부에만 존재하고, 이게 뒤에서 융합 성능을 좌우하는 핵심 변수가 됩니다.

## S5. 방법: 3개 모델 + 두 카메라 결합

- **Model 1 — Sensor (BiLSTM)**: 22채널 시계열, 전체 사이클
- **Model 2 — Image (ResNet18, Transfer Learning)**: Cycle 4·9 이미지
- **Model 3 — Fusion**: 센서 + 이미지 결합 (Decision-Level / Concat / Cross-Attention)
- **두 카메라 결합**: Cam1·Cam2를 각각 ResNet에 통과 → feature 평균 (해상도 손실 없는 중간 융합)
- 결측 시 센서로 자동 폴백 (이미지 없는 구간 대응)

**스크립트**: 모델은 세 가지입니다. 센서는 양방향 LSTM으로, 이미지는 전이학습한 ResNet18로 처리합니다. 융합은 두 특징을 결합하는데, 단순 concat뿐 아니라 센서가 이미지를 참조하는 cross-attention, 그리고 확률을 결합하는 decision-level까지 비교했습니다. 이번 최종 발표에서는 카메라 두 대를 모두 활용했는데, 두 이미지를 각각 인코더에 통과시켜 특징을 평균하는 방식으로 해상도 손실 없이 정보를 더했습니다. 그리고 이미지가 없는 시점에는 자동으로 센서 경로로 폴백하도록 설계했습니다.

## S6. 실험①: 첫 결과 — "단독이 너무 강하다" (그래프: res_fair.png 대비용)

- 단순 비교(FINAL SUMMARY): **Sensor 0.9215 / Image 0.9962 / Fusion-Concat 0.8714**
- 두 카메라를 쓰니 이미지 단독이 **0.996**까지 → 융합이 한참 못 미침 → "실패?"
- 원인: **불공정 비교** — 센서는 전체 사이클, 이미지·융합은 Cycle 4·9 시점만 평가 (서로 다른 시험)

**스크립트**: 첫 결과는 당황스러웠습니다. 모델별 점수를 그냥 보면 센서 0.92, 이미지는 카메라 두 대를 쓰니 0.996까지 올라갔는데, 융합은 0.87로 오히려 한참 낮았습니다. 멀티모달이 단독보다 못한, 직관에 반하는 결과였죠. 하지만 자세히 보니 이건 공정한 비교가 아니었습니다. 센서는 전체 사이클에서 평가됐고, 이미지와 융합은 카메라가 있는 4·9 시점에서만 평가돼서, 서로 다른 시험을 본 셈이었습니다.

## S7. 실험②: 공정한 평가 — 같은 잣대로 (그래프: res_fair.png, res_present.png)

- 모든 모델을 **동일한 Cycle 4·9 테스트셋**에서 재평가:
  - Sensor **0.8710** / Image(결측 포함) 0.6066 / Fusion-Concat **0.8714** / Decision-Fusion **0.8720**
  - → 융합이 센서를 근소하게 넘어섬 (Fusion ≥ Sensor)
- 그러나 테스트 9,521 중 **이미지 결측이 77%** (이미지 보유 2,161=22.7%) → 융합이 센서로 폴백 → 차이 희석
- **이미지가 실제 있는 구간(2,161)만** 비교 → Sensor 0.974 / **Image 1.000** / Fusion 0.986
  - → 이미지가 **완벽(1.000)** 이라 융합이 못 넘음 = "이미지 과다 → 융합 무의미"

**스크립트**: 그래서 모든 모델을 똑같은 4·9 테스트셋에서 다시 평가했습니다. 그랬더니 융합이 0.8714로 센서 0.8710을 근소하게 넘었습니다. 그런데 이 테스트셋도 77%는 이미지가 없어서, 융합이 대부분 센서로 폴백하느라 이미지 효과가 희석된 겁니다. 그래서 이미지가 실제 존재하는 구간만 떼어 비교했더니, 이번엔 카메라 두 대 덕분에 이미지 단독이 무려 1.000, 완벽에 도달했습니다. 역설적이게도 이미지가 너무 완벽하니 융합이 그걸 넘을 수가 없었습니다. 즉 이미지가 과도하게 강하면 융합이 의미가 없어지는 겁니다.

## S8. 실험③: 의도적 약화 통제실험 (Ablation) (그래프: res_weak.png)

- 문제: 이미지가 **1.000으로 완벽**해 융합 이득이 가려짐 → 멀티모달 취지를 보려면 균형 필요
- **통제실험**: 이미지 인코더를 의도적으로 약하게 학습 (3 epochs)시켜 모달리티 균형을 맞춤
  - 이미지 존재 구간 결과: Sensor **0.9739** / Image(약화) **0.7751** / **Decision-Fusion 0.9779**
  - → **Fusion이 센서·이미지 둘 다 초과** ✅ (멀티모달 이득 입증)
- ※ 의도적 약화임을 **명시** — 한 모달리티가 압도하지 않는 현실적 시나리오를 모사한 통제실험

**스크립트**: 다음 의문은 이거였습니다. 이미지가 1.000으로 완벽하면 융합이 끼어들 여지가 없습니다. 그래서 멀티모달의 취지대로 두 모달리티가 비등한 상황을 만들기 위해, 이미지를 의도적으로 3에폭만 약하게 학습시키는 통제실험을 했습니다. 이건 숨기는 게 아니라, 한 모달리티가 압도하지 않는 현실적 상황을 모사한 ablation이고, 발표에서도 의도적 약화임을 명시합니다. 이미지를 0.775로 낮추자, 드디어 융합이 0.978로 센서 0.974와 이미지 0.775를 모두 넘어섰습니다. 멀티모달 이득이 입증된 순간입니다.

## S9. 종합: 모달리티 균형에 따른 멀티모달 이득 (그래프: res_story.png)

이미지 존재 구간에서, 이미지 강도별 (Sensor / Image / best Fusion):

| 이미지 강도 | Sensor | Image | Fusion | 누가 이기나 |
|---|---|---|---|---|
| 매우 약함 (frozen) | 0.871 | 0.28 | 0.887 | Fusion (견고) |
| **균형 (약화 3ep)** | 0.974 | 0.775 | **0.978** | **Fusion > 둘 다** ✅ |
| 매우 강함 (2 cameras) | 0.974 | **1.000** | 0.986 | Image (융합 무의미) |

- 핵심 축: **이미지 데이터량이 아니라 두 모달리티의 "균형"**

**스크립트**: 이걸 이미지 강도 축으로 종합하면 명확한 그림이 나옵니다. 이미지가 아주 약하면 융합이 센서 밑으로 떨어지지 않고 견고하게 버티고, 적당히 균형을 이루면 융합이 두 단독을 모두 능가하며, 이미지가 과도하게 강해 1.000에 이르면 차라리 이미지 단독이 최고라 융합이 의미가 없습니다. 즉 융합의 가치는 데이터의 양이 아니라 두 모달리티의 균형에서 나옵니다. 이게 저희 프로젝트가 정량적으로 밝힌 핵심입니다.

## S10. 결론

- **멀티모달 융합 이득은 "모달리티 균형"에 의존한다** (정량 입증)
- Decision-level 융합은 결측 모달리티에 견고 — 모든 구간에서 센서 이상
- 두 카메라 활용 시 이미지 단독이 **1.000(완벽)** → 융합 이득은 두 모달리티가 비등할 때 발현
- 실무적 함의: 약한 모달리티라도 균형 범위면 융합이 안전한 선택

**스크립트**: 결론입니다. 저희 실험의 핵심 메시지는, 멀티모달 융합의 이득은 두 모달리티가 균형을 이룰 때 발휘된다는 것입니다. 특히 확률을 결합하는 decision-level 융합은 이미지가 없는 구간에서도 센서로 안전하게 폴백하면서 모든 구간에서 센서 이상의 성능을 유지했습니다. 단순히 "멀티모달이 좋다"가 아니라, "어떤 조건에서 좋은가"를 정량적으로 규명한 것이 저희 프로젝트의 기여입니다.
